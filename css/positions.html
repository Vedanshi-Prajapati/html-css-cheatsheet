POSITION PROPERTY

The position property specifies the type of positioning method used for an element.

There are five different position values:

1. Static
  • HTML elements are positioned static by default.
  • Static positioned elements are not affected by the top, bottom, left, and right properties.
  • An element with position: static; is not positioned in any special way; it is always positioned according to the normal flow of the page.
Example Of static
2. Relative
  • An element with position: relative; is positioned relative to its normal position.
  • The top, right, bottom, and left properties are used to offset the element from its original position.
  • Other content will not be adjusted to fit into any gap left by the element.
Example Of relative
3. Absolute
  • An element with position: absolute; is positioned relative to the nearest positioned ancestor.
  • f an absolute positioned element has no positioned ancestors, it uses the document body, and moves along with page scrolling.
  • Absolute positioned elements are removed from the normal flow, and can overlap elements.
Example of absolute
4. Fixed
  • An element with position: fixed; is positioned relative to the viewport, which means it always stays in the same place even if the page is scrolled.
  • The top, right, bottom, and left properties are used to position the element.
  • Other elements act like it doesn't exist at all â€” no space is reserved for it. It's drawn on top of the page content and stays pinned to the viewport.

5. sticky
  • An element with position: sticky; is positioned based on the user's scroll position.
  • A sticky element toggles between relative and fixed, depending on the scroll position. It is positioned relative until a given offset position is met in the viewport - then it "sticks" in place (like position:fixed).
  • You must specify at least one of top, right, bottom or left for sticky positioning to work.
